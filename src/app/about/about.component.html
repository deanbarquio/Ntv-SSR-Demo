CSR vs SSR: Complete Comparison

Understanding the fundamental differences between Client-Side Rendering (CSR) and Server-Side Rendering (SSR) in modern web development

What is the Difference?

Client-Side Rendering (CSR) renders the entire application in the browser using JavaScript. The server sends a minimal HTML file, and the browser downloads JavaScript to build the page.

Server-Side Rendering (SSR) renders the initial HTML on the server and sends it to the browser. The browser then "hydrates" the page to make it interactive.

   index: false
}));

     .then((response) => {
       response ? writeResponseToNodeResponse(response, res) : next();
     })
     .catch(next);
});

üìä CSR Data Fetching

@Injectable()
export class ProductsService {
   constructor(private http: HttpClient) {}
   
   getProducts(): Observable<Product[]> {
     return this.http.get<Product[]>('/api/products');
   }
}

// In component

   this.productsService.getProducts()
     .subscribe(products => this.products = products);
}

Flow:

  • HTTP request made after component init
